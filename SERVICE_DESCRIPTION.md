---

# 1. 서비스 개요

## 서비스명
Best of Growth Log

## 설명
그로스로그 커뮤니티에서 진행하는 성장일지 작성 경쟁 클럽의 운영을 위한 웹 서비스.

참가자는 지정된 플랫폼(Velog, Tistory, Medium)에 성장일지를 작성하고 해당 게시물의 **좋아요와 댓글 수를 기반으로 점수가 산정**되며, 리더보드에 반영된다.

# 3. 참가 규정

## 계정 생성 규칙

참가자는 반드시 **성장일지 전용 신규 블로그 계정**을 생성해야 한다.

허용 플랫폼

- Velog
- Tistory
- Medium

신청 시 등록한 계정은 다음 조건을 만족해야 한다.

### 조건

1. 플랫폼이 Velog / Tistory / Medium 중 하나여야 한다
2. 계정의 상태가 아래와 같아야 한다

- 게시물 수 = 0
- 팔로워 수 = 0
- 팔로우 수 = 0

기존 계정 사용은 **불가**

> 팔로워 수 차이로 인한 불공정 방지

# 4. 채점 방식

참가자는 **성장일지 게시물 링크를 제출**한다.

시스템은 해당 게시물의

- 좋아요 수
- 댓글 수

를 수집하여 점수를 계산한다.

---

## 점수 계산 규칙

| 항목 | 점수 |
|---|---|
| 좋아요 | 1개 = 1점 |
| 댓글 | 1개 = 5점 |

### 댓글 규칙

- 동일 인물이 같은 게시물에 여러 댓글 작성 시
- **1회만 카운트**

---

## 점수 반영 방식

게시물 링크 제출 시

→ 게시물 데이터 수집  
→ 점수 계산  
→ 리더보드 반영

---

## 리더보드 업데이트

리더보드는 하루 **2회 갱신**

업데이트 시간

- 오전 12:00
- 오후 12:00

# 9. 성장일지 제출 기능

사용자는 **차수별 성장일지 제출**이 가능하다.

---

## 9.1 성장일지 리스트 화면

리스트 형태로 표시

예시

- 성장일지 1차
- 성장일지 2차
- 성장일지 3차
- 성장일지 4차
- 성장일지 5차
- 성장일지 6차

단 **관리자가 활성화한 차수만 표시**

---

## 9.2 제출 방식

리스트 카드 클릭 시

→ 제출 페이지 이동

가능 기능

- 링크 등록
- 링크 수정

---

## 제출 검증

성장일지 제출 시 다음 조건 검증

### 조건1

게시물이 **참가 신청 시 등록한 계정에서 작성된 글인지**

### 조건2

게시물 작성일이 **제출 시작일 ~ 제출 마감일 사이인지**

주의

검증 기준

- 게시물 **작성일**
- 업데이트 날짜 아님

작성일만 조건에 맞으면 제출 자체는 나중에 진행해도 허용

# 12. 리더보드

리더보드는 다음 정보를 표시한다.

- 참가자명
- 총 점수
- 좋아요 수
- 댓글 수
- 제출한 성장일지 수

정렬 기준


총 점수 DESC

## 12.1 참가자 상세 정보 조회

리더보드에서 참가자 정보를 클릭하면  
해당 참가자가 제출한 성장일지 상세 정보를 확인할 수 있어야 한다.

표시 방식

- 다이얼로그(Dialog) 형태로 표시

---

### 다이얼로그 표시 정보

다이얼로그에는 다음 정보가 표시된다.

참가자 정보

- 참가자명
- 총 점수
- 총 좋아요 수
- 총 댓글 수

---

### 성장일지 목록

참가자가 제출한 성장일지 리스트를 표시한다.

각 성장일지는 다음 정보를 포함해야 한다.

| 항목 | 설명 |
|---|---|
| 성장일지 차수 | 예: 성장일지 1차 |
| 게시물 링크 | 원본 블로그 게시물 |
| 좋아요 수 | 해당 게시물의 좋아요 수 |
| 댓글 수 | 해당 게시물의 댓글 수 |
| 점수 | 계산된 점수 |

점수 계산 방식


score =
(좋아요 수 × 1)
+
(댓글 수 × 5)


---

### 성장일지 바로가기

각 성장일지에는 **"게시물 보기" 버튼**이 있어야 한다.

버튼 클릭 시

→ 새 탭에서 해당 블로그 게시물이 열린다.

---

### 표시 예시


참가자 : 홍길동
총 점수 : 132점

성장일지 1차
좋아요 : 10
댓글 : 3
점수 : 25
[게시물 보기]

성장일지 2차
좋아요 : 20
댓글 : 4
점수 : 40
[게시물 보기]
